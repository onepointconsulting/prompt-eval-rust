## The Loop in Plain Words

```
  Define what good looks like  (rubric)
          ↓
  Write the prompt template
          ↓
  Generate realistic test cases  (using the rubric as a guide)
          ↓
  Run: Haiku answers  →  Sonnet judges each answer against the rubric
          ↓
  Read the scores — where does it fail and why?
          ↓
  Fix the prompt  →  run again  →  did the score improve?
```

## The Two AI Roles

```
  ┌─────────────────────────────────────────────────────────┐
  │  Claude Haiku   →   the RESPONDER                       │
  │                                                         │
  │  Cheap, fast. Receives the filled prompt + question.    │
  │  Produces the response being evaluated.                 │
  │  Runs once per question per evaluation.                 │
  └─────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────┐
  │  Claude Sonnet  →   the JUDGE + GENERATOR               │
  │                                                         │
  │  More capable. Used for tasks requiring reasoning:      │
  │  · Scores responses against the rubric  (evaluation)   │
  │  · Generates prompt templates           (once)         │
  │  · Generates test case datasets         (once)         │
  └─────────────────────────────────────────────────────────┘
```

---

## What Lives Where

```
  Prompt Template     →  stored in database  (prompts table)
  Test Case Dataset   →  stored in database  (datasets + questions tables)
  Evaluation Results  →  stored in database  (evaluation_runs + evaluation_details)
  All AI calls        →  live  (nothing is cached between runs)
```
